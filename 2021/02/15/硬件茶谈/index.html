硬件茶谈 up 主视频 + 零碎硬件知识

CPU

SiO2 -> Si -> 元晶圆 -> wafer -> die

多核 cpu 性能优于 多 cpu, 但价格贵

频率

外频

主板晶振 BCLK, 100MHz

倍频

cpu 频率相对于外频翻倍

超频

超频一般超倍频, 因为改变外频需要其他所有设备同步超频

内存外频异步工作, 内存频率变为外频的 133 整数倍 Ratio, 一般这个Ratio有100 : 100和100 : 133两种选择

显卡

显卡核心频率和显存频率不受基础频率影响, 可以随意设置

总线

串并行

趋势: 并行->串行

并行: 多个数据同时传输, 需要做到整齐同步, 速度慢

PCIe

ISA 总线

上世纪 80 年代, 并行总线

PCI

即插即用, 并行传输, 共享总线, 不支持热拔插(带电插拔)

PCIe

  • 存在形态: 接口(插显卡, 网卡, 声卡等等, 转 M2, USB 等等), 通道(M2, 雷电 3)
  • 残血 PCIe*2 满血 PCIe*4
  • PCIe*4 -> 4GB/s , M2 速度不会超过 4GB/s

南桥 south bridge

帮助 cpu 和其他部分交换数据

直连 cpu 的 PCIe 只有 16 条, 所以插一个显卡 PCIe 就全部被占用了

cpu 和南桥(DMI3.0 其实就是 PCIe3.0*4)上限是 4GB/s

存储器

概述(csapp)

分类

  • RAM

    • DRAM -> 内存

      • FPM DRAM, EDO DRAM, SDRAM, DDR SDRAM, VRAM

    • SRAM -> cache

    • ROM

      • PROM, EPROM, EEPROM, FLASH

      • SSD

        U盘=USB控制器+闪存控制器+FLASH
        固态硬盘=硬盘控制器+FLASH
        FLASH一般就是各种闪存芯片，一般都是NAND的片子。

  • 磁盘, 光盘, 磁带

    • I/O 总线和接口

      USB(universal serial bus), SCSI, SATA 等

    • DMA 设备读写数据不需要 cpu 参与

类型 单元大小 管理器
CPU 寄存器 4/8B 一个字 编译器
高速缓存 64B 一个 block 硬件
虚拟内存 4KB 一个 page 硬件+OS
磁盘 512KB 等 一个扇区 磁盘控制器

cache

映射: 内存地址 -> 组内标记 + 组号 + 块偏移
组号在中间是为了防止多个地址对应同一个缓存组

分类: 直接映射, 组相联映射, 全相联

硬盘 HDD(hard disk drive)

  • 机械硬盘: 写数据直接覆盖, 删除 FAT 表
  • LMR, PMR(CMR)
  • SMR 瓦楞式堆叠硬盘

fat 表

磁盘阵列

  • raid0 一个数据分成 n 份, 带宽提高 n 倍, 最不安全, 速度最快
  • raid1 自动备份, 最安全, 空间利用最低, 速度不增
  • raid2 海明校验, 在 raid0 基础上增加纠错功能
  • raid3 raid0 + 一个存储恢复码的磁盘, 数据拆分
  • raid4 raid0 + 一个存储恢复码的磁盘, 数据不拆分
  • raid5 raid34 可能会因为恢复盘而制约性能, raid5 把恢复码分散到所有磁盘中
  • raid6 raid345 只能容灾一个磁盘, raid6 在 raid5 基础上使用两个容灾磁盘

固态硬盘 SSD(solid state disk/drive)

lba logical block address

固态硬盘: TRIM 回收指令, 在硬盘空闲时会自动执行, 固态硬盘在写入前必须进行擦除操作

层次

die -> plane -> block -> page

主控每个通道连接一个 Die, 写入时类似 raid0, 多通道提高带宽

写满后:
在SSD主控制器看来，这里的文件都是有效的，也就是不能擦除. 在操作系统的要求下，继续写入数据的话就要在空余的地方写入，对于SSD来说，这时候就需要把整个块的绿色读进缓存并更新操作系统发来的数据，之后擦除这整个块并把缓存里的整个更新完的块写回去，这样即使写入4KB的数据，其实也是大费周章的折腾，这就是早期SSD主控制器或者U盘写入放大高的道理，这些SSD的算法上不支持GC，也没有Trim,所以在随机写入的表现上非常的悲剧。

gc garbage collection

垃圾回收的功能是将所有Block中的有效Page合并到一个新的Block中，并将旧的Block进行擦除，这样做的好处一方面减少寻址负担，另一方面留出更多的空闲Block。

接口, 协议, 总线

内存

双通道

内存带宽 = 内存核心频率 * 内存总线位宽 * 倍增系数

IMC(memory controller i/o) 最大只支持双通道内存

性能制约

  • 标注内存频率为保底频率
  • 主板不超频时, 内存下限由内存本身决定, 内存上限由 cpu 默认内存频率和内存本身体质决定
  • 主板超频时内存频率无视 cpu 默认内存频率, 内存下限由内存本身决定, 内存上限由内存体质, IMC 性能/体质, 主板布线和电气性能决定

灰阶响应时间

  • PWM 调整占空比 OLED 低频 PWM

OLED

发光二极管, 通电越多越亮, 每个子像素都是独立控制开关

  • 寿命短: 有机物+频繁电子迁移+自发光, 烧屏
